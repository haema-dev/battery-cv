---
description: 구체적인 프로젝트 컨텍스트와 최신 정보를 기반으로 한 코드 검토 프로세스
---

이 워크플로우는 코드를 검토할 때 단순한 분석을 넘어, 외부 지식과 내부 구조를 모두 고려한 심층 검토를 수행하기 위해 사용합니다.

1. **최신 정보 및 베스트 프랙티스 검색 (Web Search)**
   - `search_web` 도구를 사용하여 사용 중인 라이브러리(예: Anomalib, PyTorch 등)의 최신 버전 이슈, 권장되는 사용 패턴, 또는 발생한 에러 메시지에 대한 해결책을 검색합니다.
   - 공식 문서나 커뮤니티(GitHub Issues, Stack Overflow)의 해결 방안을 참조하여 현재 코드가 최신 표준을 따르는지 확인합니다.

2. **라이브러리 참조 및 프로젝트 내 코드 연결성 분석 (Context & Library Reference)**
   - `grep_search` 또는 `list_dir`을 사용하여 현재 수정하려는 코드가 프로젝트 내 다른 파일에서 어떻게 호출되거나 사용되고 있는지 확인합니다.
   - 관련 라이브러리의 구현 세부 사항이 궁금할 경우 프로젝트 내의 참조 코드나 임포트된 모듈의 구조를 파악하여 일관성을 유지합니다.

3. **문법 확인 및 로직 검증 (Syntax & Logic Validation)**
   - `run_command`를 사용하여 문법 오류(예: `python -m py_compile [file]`)를 확인하거나, 가능한 경우 단위 테스트를 실행합니다.
   - 데이터 흐름, 예외 처리, 자원 관리(GPU 메모리 등) 측면에서 논리적인 결함이 없는지 최종 검토합니다.

위 과정을 모두 마친 후, 발견된 개선 사항이나 검토 결과를 사용자에게 보고합니다.
